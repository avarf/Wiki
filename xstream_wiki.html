The XStream programming model borrows heavily from a programming paradigm called 'reactive programming', as popularized by libraries such as Rx. Again, contrary to these existing libraries, where all code runs in a single environment, XStream script code is automatically spread across multiple execution environments. Hence, our slogan "author centrally, deploy distributed".

Env Setup

Installing TypeScript:
1- Installing node: going to its website for instalation
2- Installing TypeScript:
$ npm install -g typescript
3- Checking the instalation and also running the compiler:
$ tsc [command]

TypeScript Basics

Creating TypeScript file:

The TypeScript file extension is ".ts": [file-name].ts

Compiling TypeScript file:

$ tsc [file-name].ts
This creates a "[file-name].js" file in the same directory and we can use/run that file either in node or browser:
Running it in node:
$ node [file-name].js

Xstream Basics

Flow:
A flow defines a stream processing pipeline where data flows from an arbitrary number of input sources (where the data arrives) to an arbitrary number of output sinks (where the data leaves).
