Differences:

  • List VS Tuple:

    1. A tuple is immutable whereas a list is mutable
    2. Tuples are fixed size in nature whereas lists are dynamic
    3. Tuples are faster than lists. If you're defining a constant set of values and all you're ever going to do with it is iterate through it, use a tuple instead of a list
    4. Some tuples can be used as dictionary keys but lists can never be used as dictionary keys, because lists are not immutable
    5. You can't add elements to a tuple. Tuples have no append or extend method
    6. You can't remove elements from a tuple. Tuples have no remove or pop method
    7. You can find elements in a tuple, since this doesnt change the tuple. You can also use the in operator to check if an element exists in the tuple.

Functions

Define a Function:


def my_function(arg1, arg2):
"FUNCTION DEFINITION AND RETURN"

Argumants:


  • Required arguments:
    These are normal arguments that we use such as: my_function(arg_1 , arg_2)
  • Default arguments
    These are arguments with some default values so if we call them without these arguments the function will use the default value: my_function(arg_1, arg_2, age = 50)
  • Variable-length argument:
    There are situations that we don't know how many arguments we will send to an argument and we have to use this type of arguments and we define them by an "*" before the name of the argument and we work with them like a tuple: my_function(arg_1, arg_2, *var_arg)

Pass by reference:


In python everything is pass by reference which means if we change that argumant in a function it will change globaly. The only thing that we must notice is if we declare a new variable inside the function then that would act as a local variable and the changes wouldn't affect that variable outside the function(Offline python application)
